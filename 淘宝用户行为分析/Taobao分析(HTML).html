提出问题¶

1.分析用户在不同时间下的行为规律，探索用户在不同时间周期下的活跃规律
2.分析用户对不同种类商品的偏好，找到针对不同商品的营销策略
3.分析用户使用淘宝中的常见电商分析指标，确定各个环节的流失率，找到需要改进的环节
4.分析用户结构，对不同用户进行价值分析

用户对不同商品偏好的结论¶

在购买量前10的的商品中只有1个浏览量前10的商品，而在收藏量前10中却有4个位于浏览量前10的商品，这说明这些吸引了用户注意力的商品并不能直接转化为销量，而是更多的被收藏。
加入购物车和购买的重复商品有2件，这说明加入购物车比浏览转化成实际销量的可能性更大(符合常识) 而收藏和购买没有重复的商品，这说明加入收藏之后大概率不会再购买。
针对浏览量高而销量不高的这部分商品，需要提高的是用户从点击进入商品详情页到最终购买的体验。作为商家端可以从以下几个方面提高销售额：
（1） 商品详情页的实际价格是否相比展示价格偏差过大，杜绝商家为了吸引用户点击在商品展示页投放的价格与实际价格不符，在用户心中反而引起抗拒心理。
（2）同时也要杜绝商家在商品首页大搞噱头而与实际商品质量或观感不同的行为。
（3）对收藏进行优化，定期提醒用户收藏内的商品，以免用户在加入收藏后对之抛掷脑后。
（4）优化商品展示的形式，利用视频等方式给用户更直观的感受，提高照片的美观程度。

基于AARRR理论分析结论¶

  1. 获取用户:
    由于没有用户首次登陆的时间，因此默认所有用户都为首次登陆的新用户。
    这里新用户一共有37376名
  2. 激活用户:
    由于数据时间跨度不够完整，因此这里将各种行为次数在50次以上用户定义为活跃。
    活跃用户一共有25054名，活跃用户比例占67%
    用户从点击、放进购物车、收藏以及购买，由于收藏和加入购物车都为浏览和购买阶段之间确定购买意向的用户行为，因此将其算作一个阶段，而从买前阶段到购买行为的行为转化率只有2%。 而从买前阶段到有购买行为的客户数转化率达到32%，当然有一部分用户是直接购买，所以为了进一步提高这一转化率，特别是针对行为转化率可以：
    （1）优化电商平台的搜索匹配度和推荐策略，主动根据用户喜好推荐相关的商品，提升用户的购买欲。
    （2）对于再收藏中放置时间比较定期像客户发送提醒，防止用户将商品放入收藏之后就抛之脑后。
    （3）优化加入购物车的体验，简化步骤的同时提升用户的预购买满足感(比如加一些动效等等)
  3. 提高留存率
    数据中只有1名用户再首次登陆后再未进行过登录，这说明淘宝的用户留存率是很高的
  4. 获取收入
    可以看出淘宝的PUC非常高,达到了68%。想要进一步扩大购买用户占比则应当增加从买前阶段到购买行为的行为转化率， 比如 : (1) 对付费意愿低的用户可以推荐性价比的商品 （2）对付费意愿低的用户可以进行促销活动，发放代金券等
  5. 用户推荐: 鉴于淘宝的用户普及程度，这里认为对于一二线城市推广可以减少，转而对三四线城市可以加大推广力度，例如和三四线城市的实体店进行合作扩大用户范围。

def Avg_distribution(df):
    dfed=df
    x=dfed.shape[0]
    begin=0
    for i in range(9):
        dfed.iloc[begin:begin+x//10,1]=i
        begin+=(x//10)
    return dfed
bb=buy_table[buy_table['User'].isin(one_buy['index'])]
bb['Diff']=9

用户评估分数¶

  • R(评分) :
    统计的时间跨度在9天，因此R值的分数可以用10减去时间间隔，即两次购买间隔为0天的用户为10分，间隔1天为9分间隔越大分数越低，最低为1分。
  • F(评分) :
    购买次数在1-9次得1分，10-19次得2分，20-29次得3分，30-39次得4分，40-49得5分，50次以上得6分
  • M(评分) :
    由于没有商品的单价信息，无法计算M值，所以都当作0。
  • 总分 :
    由于时间跨度比较小，即使R评分为9分即用户每天最少都购买一次也有可能只购买了9次。所以这里我认为结合数据集来看R评分的重要程度＜F评分的重要程度。因此设置F评分占总分权重60%,R评分占40%。

def change(x):
    if x<=9:return 1
    if 10<=x<=19:return 2
    if 20<=x<=29:return 3
    if 30<=x<=39:return 4
    if 40<=x<=49:return 5
    if x>=50:return 6

R_table

score_table["score"]=score_table['R']*0.4+score_table['F']*0.6
score_table=score_table.sort_values(by='score',ascending=False).reset_index(drop=True)

结论¶

综合评分越高的用户是体系中的越有价值用户，对于不同类型的用户需要采取不同的策略。
对于潜在用户，运营活动可以重点针对这部分用户，可以通过拼团打折、积分兑换等活动唤起用户购买欲望。
对于活跃和成熟用户则应用更稳健的手段，以优化用户使用体验为主，辅助以一定的个性化推送提醒。
对于核心客户需要给予一定的关注，可以设置专门的反馈渠道进一步优化这类用户的体验。
